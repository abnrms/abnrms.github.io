UNO Card Game

Java (Swing)

This project is a GUI-based card game written in Java using Swing to handle GUI functionality. The game mimics the behavior of the popular game UNO, handling matching cards by number and color, skips, reversing the order of play, draw cards, and wild cards. This game was created as a final project in a group of 4 for my Object-Oriented Programming and Design class, and thus focused on using OOP principles and following design patterns in its creation. GitHub was used to handle version control, with myself managing the branch, handling merge conflicts and ensuring consistency on top of my coding duties.

During this project, my duties involved the aforementiong Git management, creating and writing different classes and methods for the functionality of the game, creating a UML diagram to capture object interactions, and writing thorough test cases using JUnit which ensured code coverage. Some of the Java techniques/components we employed in this project were class inheritance, enums, MVC, and the observer pattern.

Relevant Skills

  • Java: This project was coded entirely in Java, using features such as class inheritance, enums, and built-in classes.
  • Swing: The GUI component of this project was developed using Swing, handling events and drawing textures on screen.
  • UML: I created a UML diagram to show this project's class structure and guide method interactions.
  • JUnit: JUnit was used to handle testing of the project code, with a suite of testcases reaching over 93% code coverage for all files.
  • GitHub: GitHub was used for collaboration and version control, with myself handling merge conflicts and general branch management.
  • Object-Oriented Design: OOP principles and patterns were used to guide the creation of this project, including observer and MVC.
